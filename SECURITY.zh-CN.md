# 安全策略

[English](./SECURITY.md) | [简体中文](./SECURITY.zh-CN.md)

## 支持范围

项目目前仍处于早期阶段。安全修复通常会优先落到当前默认开发分支。

## 如何报告漏洞

请不要在公开 issue 中直接披露完整漏洞细节。

建议优先顺序如下：

1. 如果仓库启用了 GitHub Security Advisories 或私密漏洞上报，请优先使用。
2. 如果没有私密上报入口，请通过私下渠道联系维护者。
3. 如果以上两种方式都不可用，请仅提交一个最小化的公开 issue 来请求安全联系渠道，不要附带利用细节、凭据或具体服务器信息。

## 报告中建议包含的内容

一份高质量的安全报告通常应包含：

- 受影响的模块或文件
- 风险影响概述
- 复现步骤
- 可在私下安全共享的 PoC 或验证信息
- 版本、提交号或分支信息
- 你已尝试过的缓解方案

## 响应目标

我们的目标是：

- 在条件允许时，于 3 个工作日内确认收到报告
- 复现问题并评估严重程度
- 制定修复与验证方案
- 在补丁可用后协调披露时间

## 本项目中的敏感区域

如果问题涉及以下部分，请特别谨慎处理：

- SSH 认证或主机校验
- 远端文件写入与备份处理
- 配置拼装与回滚逻辑
- 对不可信订阅输入的解析
- 在被管理服务器上的命令执行
